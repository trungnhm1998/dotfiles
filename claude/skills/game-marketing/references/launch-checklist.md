# Launch Checklist

## Pre-Launch (4–8 Weeks Out)

### Steam Page
- [ ] All capsule images finalized (header, small, main, library)
- [ ] 10+ screenshots (mix of gameplay, UI, cinematic moments)
- [ ] Launch trailer uploaded
- [ ] Short and long descriptions finalized
- [ ] Tags finalized (15–20)
- [ ] System requirements accurate and tested
- [ ] DLC / soundtrack pages set up (if applicable)
- [ ] Mature content survey completed
- [ ] Legal notices / credits in place
- [ ] Price set (check regional price equivalents in Steamworks)

### Technical
- [ ] Build submitted and reviewed (Steam review takes 3–5 business days)
- [ ] Store / DRM settings confirmed
- [ ] Achievements implemented and tested (optional but raises engagement)
- [ ] Steam Cloud saves configured (optional but appreciated)
- [ ] Controller support verified (add the controller support tag if supported)
- [ ] Resolution / fullscreen / quit behavior working correctly
- [ ] Crash reporter integrated (optional: Sentry, GameAnalytics)
- [ ] No debug logs / dev shortcuts left in the build

### Press & Content Creators
- [ ] Press kit created: game description, screenshots, logos, trailer link, contact email. Host on **presskit.gg** (WordPress plugin, free) or a simple public Google Drive folder. (presskit() by Rami Ismail is legacy/unmaintained — avoid.)
- [ ] List of 20–50 relevant content creators researched (YouTube, Twitch, TikTok matching your genre)
- [ ] Draft outreach email written: 3 sentences, key image/GIF, Steam link, review key offer
- [ ] Steam review keys generated (Steamworks → Packages & Pricing → Create Packages)
- [ ] Key distribution platform set up (Keymailer, Woovit, or manual)
- [ ] Outreach sent 2–4 weeks before launch (coverage needs production time)

### Community
- [ ] Announcement post drafted for all platforms
- [ ] Discord / community platform ready (if applicable)
- [ ] Launch day posting schedule written out
- [ ] Relevant subreddits identified and posting rules reviewed

## Launch Day

- [ ] Flip the release: Steamworks → Release → Set Release Date → Publish
- [ ] Post announcement on all social platforms (simultaneously if possible)
- [ ] Pin an announcement in Discord
- [ ] Monitor Steam discussions and reviews hourly; respond to early reviews and discussions
- [ ] Check that store page is rendering correctly
- [ ] Verify build is downloadable and playable by a fresh account

## Launch Week

- [ ] Respond to all Steam reviews (positive and negative; be concise and friendly)
- [ ] Post a "thank you / launch day update" on social within 24h
- [ ] Fix any critical bugs reported; push hotfix build within 24–48h if needed
- [ ] Post sales numbers update (if comfortable — transparency builds trust)
- [ ] Reach out to any streamers who covered the game to thank them

## Post-Launch (Week 2+)

- [ ] Patch notes post for each update
- [ ] Roadmap post (if continuing development)
- [ ] Wishlists → sales conversion review (look at Steamworks analytics)
- [ ] Consider a launch sale (Steam sale events) 30–90 days after launch
- [ ] Start gathering testimonials / quotes for future marketing

## Buy-vs-Build Notes

- **Press kit hosting**: use presskit.gg (free WordPress plugin) or a Google Drive folder. Don't build your own. presskit() (dopresskit.com) is unmaintained and breaks on modern hosting — skip it.
- **Review key distribution**: Keymailer / Woovit handle discovery by creators automatically; manual is fine for a small launch.
- **Localization**: defer unless the genre strongly benefits (JRPGs, visual novels) — translate the UI strings only as a post-launch expansion.
