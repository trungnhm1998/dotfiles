## 1. Phase Overview

| Phase | Goal | Gating question |
|---|---|---|
| **Prototype** | Is the mechanic viable? | Does the core idea work at all? |
| **Vertical Slice** | Is the game fun? | Would a stranger want more? |
| **Production** | Build the game | Is the scope locked? |
| **Content Complete** | Everything in, rough edges remain | Can someone play the whole game? |
| **Beta / Polish** | Fix and finish | Is it embarrassment-free? |
| **Ship** | Release | Have you tested the release build? |

Do not rush to Production. The most expensive mistake is entering Production before Vertical Slice exits.

## 2. Phase Details

### Prototype
**Entry:** You have an idea and can describe the core mechanic in one sentence.

**Focus:**
- Build the fastest possible version of the core mechanic. Ugly is fine.
- No menus, no UI, no story — just the mechanic.
- Try it, hate it, iterate, or pivot.
- Time-box aggressively: days or 1–2 weeks maximum.

**Exit criteria:**
- [ ] You've played the mechanic enough to believe it's worth developing.
- [ ] You can describe what the vertical slice will prove.
- [ ] You know what the core loop is (can name it in one sentence).

---

### Vertical Slice
**Entry:** Prototype passed. Core mechanic is viable.

**Focus:**
- One polished, end-to-end fragment of the real game.
- Establish the art direction, audio direction, feel standard.
- Design and tune one real encounter/level — not placeholder.
- Playtest with real people. Iterate on feedback.

**Exit criteria:**
- [ ] Stranger can play it without your help.
- [ ] Playtesters engage with the core loop as intended.
- [ ] You are not embarrassed to show it.
- [ ] You want to keep making this game.
- [ ] Art direction is locked (or near-locked).

---

### Production
**Entry:** Vertical Slice passed. Scope is written down and locked.

**Focus:**
- Build the game's remaining content and systems against a fixed scope list.
- Hold the scope line. New ideas go to the Later list, not the sprint.
- Maintain the quality bar set by the vertical slice.
- Regular playtesting (every 4–6 weeks minimum) to catch drift.

**Exit criteria:**
- [ ] All intended content exists in-game (rough is fine, complete is required).
- [ ] All core systems implemented and functional.
- [ ] Scope list is checked off or deliberately cut.

---

### Content Complete / Feature Freeze
**Entry:** Production done. Scope is fully in-game.

**Focus:**
- No new features. No new content. Only fix and polish.
- Full playthrough without a crash or progress-blocker.
- Audio and visual pass on all content.
- Performance pass — consistent frame rate on target hardware.

**Exit criteria:**
- [ ] Game can be played start-to-finish without a blocker.
- [ ] No crashes in normal play.
- [ ] All placeholder assets replaced.
- [ ] Frame rate is acceptable throughout.

---

### Beta / Polish
**Entry:** Content complete. Game is playable start-to-finish.

**Focus:**
- External playtesting (wider audience than friends).
- Bug triage — fix high-impact bugs, document or accept low-impact ones.
- Juice, feel, and feedback polish — now is the time.
- Store page, trailer, press kit (if releasing on a platform).
- Build the release version and test it on the target platforms.

**Exit criteria:**
- [ ] No P0/P1 bugs remaining.
- [ ] 3+ external playtesters completed the game.
- [ ] Release build tested on target platforms.
- [ ] Store/distribution assets ready.

---

### Ship
**Entry:** Beta complete. Release build tested.

**Focus:**
- Final review of the release build (not the editor build).
- Submit / upload / publish.
- Have a plan for the first week of feedback: watch, note, do not panic-patch.

**Exit criteria:**
- [ ] Live.

## 3. Example Milestone Breakdown (Small Game)

Scenario: A solo dev making a small 2D action-platformer, part-time (12h/week).

| Milestone | Duration | Deliverable |
|---|---|---|
| Prototype | 2 weeks | Playable movement + one enemy type, no art |
| Vertical Slice | 6 weeks | One polished level, real art direction, 3 playtest sessions |
| Production M1 | 8 weeks | 3 levels, 2 enemy types, all core systems |
| Production M2 | 8 weeks | Remaining levels (target: 6–8 total), boss, collectibles |
| Content Complete | 3 weeks | All content in, full playthrough possible |
| Beta | 4 weeks | Bug fixes, polish pass, store page |
| Ship | 1 week | Release build + launch |

**Total:** ~32 weeks (~8 months) part-time. This is an optimistic estimate for a small game with a locked scope — build in a 20% buffer.

### Notes
- Each "Production Mx" milestone should have a written scope list reviewed at the start. Cut anything that doesn't fit in the time box.
- If Production M1 slips significantly, reassess total scope before starting M2.
- Do not start on the trailer or store page before Content Complete — the game will still change.
